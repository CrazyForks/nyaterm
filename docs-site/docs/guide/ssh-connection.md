# SSH 连接管理

## 连接配置

### 基本信息

创建 SSH 连接时，需要填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 连接名称 | 用于识别的显示名称 | 否 |
| 图标 | 自定义图标 | 否 |
| 分组 | 连接所属分组 | 否 |
| 主机 | 服务器 IP 或域名 | 是 |
| 端口 | SSH 端口号（默认 22） | 否 |
| 用户名 | 登录用户名 | 是 |
| 认证方式 | 密码或私钥 | 是 |
| 描述 | 连接的备注说明 | 否 |

### 认证方式

Dragonfly 支持两种认证方式：

#### 密码认证

直接输入服务器密码。密码会使用 AES-256-GCM 加密存储在本地。

#### 密钥认证

选择已导入的 SSH 私钥进行认证。支持 RSA、Ed25519 等常见密钥格式。

如需管理密钥，前往 **设置 → 安全 → 密钥管理**。

## 连接分组

你可以创建文件夹来组织连接：

- 右键点击侧边栏空白区域选择 **新建文件夹**
- 支持多级嵌套文件夹
- 拖拽连接到文件夹中进行分组
- 右键文件夹可重命名或删除

## 连接排序

侧边栏支持三种排序方式：

- **自定义顺序** — 手动拖拽调整
- **名称 A → Z** — 按名称升序
- **名称 Z → A** — 按名称降序

## 导入会话

Dragonfly 支持从其他 SSH 客户端导入会话：

1. 右键点击侧边栏选择 **导入会话**
2. 选择导入来源（目前支持 WindTerm）
3. 选择对应的会话配置文件
4. 确认导入

## 主机密钥验证

Dragonfly 使用 TOFU（Trust On First Use）策略管理主机密钥：

- **提示** — 首次连接时询问是否信任（默认）
- **接受** — 自动接受所有新主机密钥
- **严格** — 拒绝所有未知主机密钥

可在 **设置 → 安全 → 连接安全** 中修改策略。

## 代理支持

如需通过代理连接，前往 **设置 → 代理** 配置：

- 支持 SOCKS5 协议
- 配置代理主机和端口
- 启用/禁用代理
